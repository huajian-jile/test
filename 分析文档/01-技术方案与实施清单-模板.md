## 三、已规划将要做的工作（可按优先级调整）

### 阶段 A — 基础能力

1. 引入 `spring-boot-starter-web`、`spring-boot-starter-validation`  
2. 引入 `spring-boot-starter-security`，实现登录、登出、密码策略（可先用 BCrypt + 内存用户，再换库表）  
3. 统一异常处理、统一返回体 `Result<T>`  
4. 集成所选 ORM 与数据库驱动；Flyway/Liquibase 管理版本化建表  

### 阶段 B — RBAC

1. 用户、角色、权限（菜单/按钮/API）数据模型与初始化数据  
2. 登录后返回当前用户菜单树、按钮权限码列表  
3. 方法级或 URL 级鉴权（`@PreAuthorize` 或自定义元数据）  

### 阶段 C — 慈善业务（对齐《需要做什么》）

1. **行政资金层级**：国家 → 省 → 市/区 → 镇/街道 → 村/社区 → 户；拨款批次与金额台账  
2. **资金流水与节点**：每笔划拨在节点上的「进入时间、转出时间、留存时长、是否被替换/冲正」等审计字段（具体规则你方定稿）  
3. **个人捐赠者**：捐赠人档案、捐赠记录、可选匿名展示规则  
4. **贫困户需求**：需求发布、状态（待审核/已对接/已关闭）、与帮扶/资金关联（若需要）  

### 阶段 D — 页面初稿（四大模块目录）

侧栏一级目录固定为四类（详见 `03-页面与RBAC功能说明-模板.md`）：

1. **国家层面**：政府拨款链、批次、划拨、节点分析等  
2. **个人慈善**：捐赠人、捐赠记录、公示等  
3. **被扶贫对象**：户档案、到户资金、需求单（及可选「我的」）  
4. **系统管理**：用户/角色/权限/组织、日志、**管理员查看明文密码协助找回**（与库表字段约定一致）  

另可选：大屏或公开查询页（脱敏展示）。  

**库表与审计**：全表统一 `created_at`、`updated_at`、`login_at`、`created_by`、`updated_by`、软删除标记；用户表 `password` 为明文（高风险，见 `02-数据库设计说明-模板.md` 安全提示）。  

### 阶段 E — 非功能

1. 操作日志、登录日志  
2. 数据权限（按区划只能看本节点及以下）  
3. 备份与灾备策略（运维文档，另附）

## 四、风险与依赖（模板占位）

- **政策与合规**：资金展示范围、脱敏规则、与财政/民政系统对接方式需业务方确认。  
- **身份源**：是否与统一身份认证（政务）对接；若对接，本系统只做权限与业务。  
- **Spring Boot 4**：生态与团队熟悉度；若不稳定可降级到 3.x LTS。
